# Sophia Stack v1.5 "Stable" — Plan

> **Theme: _Anybody can be a dev._** A polished, Lovable-class live-building experience — self-hosted
> and agent-operable — where a non-technical person goes from a downloaded zip to a live website in
> minutes, the AI builds it in front of them with a live preview, one click adds the **Sophia SEO
> Suite**, and updates are **safe and reversible**. At v1.5 the Stack and the SEO Suite **sync
> perfectly**.

**Non-negotiable for every item below: nothing is destructive.** Every update, install, and migration
backs up first, validates, and **auto-rolls-back / auto-stops on any failure**. Sensitive operations
(updates, installs) require the owner to be **logged in**.

## 1. Workstreams

### WS1 — Intuitive dashboard + first-run onboarding
**Goal:** a guided, friendly control panel; a non-technical owner is never lost.
- **Onboarding wizard** (first run, step-by-step, dismissible, resumable): ① create admin → ② save the
  5-word recovery phrase (confirm) → ③ connect an AI provider (one-tap presets) → ④ pick a template or
  describe your site → ⑤ you're live. A persistent **"Setup checklist"** card shows remaining steps.
- **IA cleanup:** consistent nav, clear section headers, empty states with a next action, inline help
  tooltips, friendly error messages, mobile-friendly layout.
- **Settings = Basic by default + an _Advanced_ toggle.** Basic: AI key, site brief, payments connect,
  updates. Advanced (hidden until toggled): base URLs / model / adapter type, env-provider list,
  extensions management, update channel + auto-update policy, OAuth, audit log, sandbox notes, raw
  token management.
- **Acceptance:** fresh deploy → guided to a live site in ≤ 30 min with zero docs; Advanced hides
  complexity from normal users.

### WS2 — Lovable-class live build (VEX in the chat pane)
**Goal:** the in-app build experience looks and feels like Lovable/Bolt — chat on one side, the site
rendering live on the other, push-to-live.
- **Streaming + status:** stream the agent's reply, show "building…/editing <block>…" states, and a
  compact summary of what changed (inline diff chips).
- **VEX upgrades:** multi-change staging (a running list of pending edits with per-item Apply/Discard),
  "Push to Live" for the whole batch, device-width toggle, and the preview front-and-center.
- **Agent compatibility (BYO + external):** the in-app builder works with **any** provider (Codex via
  OpenAI-compatible, Claude, Grok, Gemini, local); **external agents (Claude Code, Codex, Grok build,
  Cursor)** can drive the same site over REST/MCP/OpenAPI and their edits appear live in VEX. Document
  a "build with your agent + watch it in VEX" flow for each.
- **Acceptance:** type a request → watch it render live → push to live, smooth and obvious.

### WS3 — Safe auto-update (Settings → Check for updates)
**Goal:** one-click updates that **can't brick a deployment** and surface what changed.
- **Settings → "Check for updates"** (Basic): shows current vs latest; **auto-pull** when enabled.
- **Non-destructive apply pipeline:** require **logged-in owner** → snapshot **code + data** → download
  + **verify checksum/signature** → swap code → **forward-migrate data** → **health-check** the new boot.
- **Auto-stop / rollback:** if download/verify/migrate/health-check fails at any point, **abort and
  restore** the previous code + data automatically; never leave a half-applied state.
- **Changelog on completion:** when the update succeeds, show the released **changelog** in-dashboard.
- **Acceptance:** updating is one click; a bad release self-reverts; the owner sees the changelog.

### WS4 — One-click Sophia SEO Suite install
**Goal:** add the SEO Suite from the dashboard with a single click — pulled and auto-installed,
non-destructively.
- **Extensions tab → "Add Sophia SEO Suite":** fetch the published extension package from its release
  channel, **verify compatibility** (`requires.sophiaStack`) + integrity, install into
  `.sophia-data/extensions/sophia-seo-suite/`, enable it, and surface its admin nav — all without
  touching site data.
- **Generalized extension installer:** the same one-click installer works for any
  manifest-compliant extension from a trusted channel (install / update / disable / **uninstall with
  no data loss**).
- **Acceptance:** one click → SEO Suite installed + enabled + visible; uninstall is clean.

### WS5 — Zip runs out of the box (self-contained auto-deploy)
**Goal:** download the zip → it just works, with everything it needs.
- **Fully self-contained artifact:** `app.js` (zero-install) + client + catalog + templates bundled;
  first boot runs onboarding. Honors `PORT` (TCP or socket).
- **`start` launcher:** picks a free port, prints the URL (opens a browser where possible), and lands on
  the onboarding wizard. A README-in-zip with the 3-step quickstart.
- **Optional bundled SEO Suite:** ship a pinned SEO-Suite package in the zip (or fetch on first run) so
  "download → run → add SEO in one click" works offline-friendly.
- **Acceptance:** unzip → `node app.js` (or `./start`) → working site + onboarding, no extra steps.

### WS6 — Stack ⇄ SEO Suite version sync (the 1.5 contract)
**Goal:** both systems interlock perfectly at 1.5.

## 2. Non-destructive specification (applies to WS3 + WS4 + updates/installs)

1. **Auth gate:** the owner must be logged in; the action is audited.
2. **Snapshot first:** code backup (previous artifact) + data backup (state files) before any change.
3. **Verify:** integrity/checksum (and signature when available) of any downloaded artifact.
4. **Apply forward-only:** swap code; run **additive** migrations; `.sophia-data` is never deleted.
5. **Health-check:** the new state must boot + pass a self-check before it's accepted.
6. **Auto-stop / rollback:** any failure restores the prior code + data and reports the reason.
7. **Visible result:** success shows the changelog; failure shows what happened and that nothing changed.

## 3. Phasing

- **Phase A — Foundations of safety + UX:** onboarding wizard + Settings Basic/Advanced toggle (WS1);
  safe-update apply/rollback/health-check + changelog + Settings "Check for updates" (WS3).
- **Phase B — One-click + out-of-the-box:** generalized one-click extension installer + SEO Suite button
  (WS4); self-contained zip + `start` launcher (WS5).
- **Phase C — Lovable-class build:** streaming chat, multi-change VEX staging, agent-compat docs for
  Codex/Claude Code/Grok (WS2); ship R3/R4/R5 (WS6).
- **Phase D — Sync + polish + cut:** screenshots/GIF in both READMEs (WS7); compatibility matrix; joint
  **v1.5.0** release (auto-update channel goes live).

Each phase: built incrementally, tests green, audit-clean, Sophia-branded commits, fresh zip delivered.
